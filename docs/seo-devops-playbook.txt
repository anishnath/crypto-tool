High‑CTR + SEO Checklist (Per Page)
-----------------------------------
1) Title
- Preferred pattern: "<Primary Keyword/Task> – <Benefits/Modifiers> – Free | 8gwifi.org"
- Keep under ~60–65 chars when possible; include “Free/Online/No Signup” where it fits.
- Ensure H1 matches the primary intent of the title.

2) Meta Description
- 150–160 chars max. Lead with core intent + 3–4 benefits (privacy, client‑side, presets, examples, copy/download).

3) Canonical URL
- Add a canonical tag pointing to the page’s absolute URL.
  Example: <link rel="canonical" href="https://8gwifi.org/my-tool.jsp" />

4) Open Graph + Twitter
- Add OG/Twitter tags for shareability and consistent link previews.
- Required: og:type, og:title, og:description, og:url, og:site_name, og:image
- Twitter: twitter:card=summary_large_image, twitter:title, twitter:description, twitter:image
- Image fallback policy (used in this pass):
  - Use an existing, relevant image if present under images/site/ (e.g., kube.png, nginx.png, terminal.png).
  - Otherwise fallback to logo.png to avoid broken references.

5) JSON‑LD: WebApplication
- Use https schema context.
- Include: name, description, url, applicationCategory, operatingSystem, offers.price=0, author, datePublished, dateModified.
- Update dateModified whenever you edit the page.

6) JSON‑LD: FAQPage (+ Visible FAQs)
- Add 3–5 high‑quality Q&As that match page content (no generic duplication; keep 1–2 sentence answers).
- IMPORTANT: Include a visible “FAQs” section on the page mirroring the JSON‑LD to satisfy structured‑data policies.

7) Internal Links
- Add a “Related DevOps Tools” block with keyword‑rich anchors to 3–5 adjacent tools (optional but recommended).

8) Trust/UX Signals
- Above‑the‑fold badges help CTR: “Free • No signup • Client‑side • Copy YAML/Code”. Keep short.

9) JSP UTF‑8 Encoding Directive (Required)
- Ensure the very first line of each JSP includes the UTF‑8 page directive:
  <%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>
- This prevents encoding issues in meta tags and visible text.


Snippets You Can Reuse
----------------------
Canonical + OG/Twitter (drop into <head>):
------------------------------------------
    <link rel="canonical" href="https://8gwifi.org/your-page.jsp" />
    <meta property="og:type" content="website" />
    <meta property="og:title" content="Your Page – Primary Benefit" />
    <meta property="og:description" content="1–2 line value prop. Free, no signup." />
    <meta property="og:url" content="https://8gwifi.org/your-page.jsp" />
    <meta property="og:site_name" content="8gwifi.org" />
    <meta property="og:image" content="https://8gwifi.org/images/site/logo.png" />
    <meta name="twitter:card" content="summary_large_image" />
    <meta name="twitter:title" content="Your Page – Primary Benefit" />
    <meta name="twitter:description" content="1–2 line value prop. Free, no signup." />

-----------------------------------------

    </script>

Verification Commands (run from repo root)
------------------------------------------
List titles for quick audit:
    rg -n "<title>.*</title>" src/main/webapp/*.jsp

List OG/Twitter images referenced by DevOps pages and check existence:
    rg --no-filename -o 'images/site/[A-Za-z0-9._-]+\.(png|jpg|jpeg|gif)' \
      src/main/webapp/{nginx-config-generator.jsp,load-test-generator.jsp,chmod-calculator.jsp,systemd-generator.jsp,dockerfile-generator.jsp,firewall-generator.jsp,apache-virtualhost-generator.jsp,github-actions-generator.jsp,gitlab-ci-generator.jsp,websocket-client.jsp,prometheus-query-builder.jsp,curl-builder.jsp,cron-generator.jsp,helm-chart-generator.jsp,kube.jsp,kube1.jsp,kube2.jsp,dc.jsp,rbac-policy-generator.jsp,service-mesh-generator.jsp,k8s-resource-calculator.jsp,rate-limiter-generator.jsp,dc1.jsp,dc2.jsp,aws.jsp,aws-smtp.jsp,inframap.jsp} \
      | awk -F/ '{print $3}' | sort -u > .referenced_devops_images.txt

    echo "\nExistence check:";
    while read -r img; do [ -z "$img" ] && continue; \
      if [ -f src/main/webapp/images/site/$img ]; then echo OK $img; else echo MISS $img; fi; \
    done < .referenced_devops_images.txt

Quick grep for canonical and OG tags on a page:
    rg -n "canonical|og:title|twitter:title" src/main/webapp/<page>.jsp

Find JSPs missing the UTF‑8 page directive:
    rg -L -n "^<%@ page language=\"java\" contentType=\"text/html; charset=UTF-8\" pageEncoding=\"UTF-8\"%>" src/main/webapp/*.jsp

Add directive (manual): insert as the first line of the JSP file.
